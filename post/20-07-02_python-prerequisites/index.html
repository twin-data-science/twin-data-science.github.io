Ferramentas para começar com Python

2020.07.02
development python

Aqui estão descritas algumas ferramentas importantes que devem estar instaladas e funcionando na sua máquina.

Utilize uma interface para Linha de Comando (Command Line Interface)

Para desenvolver na sua máquina, você vai precisar acessar sua interface com a linha comando da sua máquina. Caso utilize macOS / linux, abra a aplicação Terminal. Se você é usuário de windows, abra o prompt de comando (‘Command Prompt’).

Obs: Para padronizar, chamarei sempre de Terminal, que é a mesma coisa que Linha de Comando / Console / Prompt de Comando.

Crie um diretório de desenvolvimento

Uma boa dica é manter seus diretórios (pastas) de desenvolvimento isolados das demais no seu computador. Gosto de criar um diretório “Developer” no diretório raíz do meu computador, para que os códigos fiquem em um local centralizado. Para fazer isso no macOS / linux / Windows, abra seu terminal e execute o comando:

$ cd ~/

para localizar o diretório raíz. Em seguida execute:

$ mkdir Developer

Este comando vai criar um novo diretório chamada Developer (geralmente lado a lado de outras pastas como “Desktop” / “Documents” / “Pictures” / “Music” / etc.). Para acessar esta pasta pelo terminal, a partir do diretório raíz e execute:

$ cd Developer

desta forma, o diretório de trabalho será agora ‘Developer’. Lá dentro você pode criar mais sub-pastas para novos projetos (exemplo: $ mkdir new-project) e acessar estes novos diretórios (exemplo: $ cd new-project).

Obs: Para padronizar, chamarei sempre de diretório (geralmente conhecido como ‘pasta’).

Escolha um bom Editor de Texto

Gosto de usar VSCodium, a versão Free/Libre Open Source Software do editor de texto da Microsoft, VSCode (o código fonte do vscode contém telemetria/rastreamento). Se não se importar, você também pode usar o VSCode ou qualquer outro editor de texto de sua preferência (Atom, Notepad++, etc.)

Instale o Python

Python é uma das linguagens de programação favoritas para Ciência de Dados. Utilize as instruções no site oficial ou um gerenciador de pacotes para baixar e instalar corretamente. Utilize Python 3.x que é versão com suporte e em desenvolvimento ativo (mesmo que o Python 2.x já venha instalado na sua máquina).

Para simplificar esta e outras instalações, recomendo utilizar o gerenciador de pacotes Homebrew (macOS / linux). Após instalação, você instalará facilmente o Python 3 executando o comando:

$ brew install python

Executar o comando $ python geralmente inicializa o interpretador de Python 2. Para verificar a instalação do interpretador de Python 3 utilize:

$ python3 --version

Caso esteja instalado, executar o comando $ python3 , inicializará o interpretador. Digite $ exit() (ou utilize o atalho Ctrl + Z no Windows | Ctrl + D no macOS / linux) para finalizar a execução do interpretador.

A maioria das distribuições do Python vem com o gerenciador de pacotes pip pré-instalado, você pode checar sua existência executando o comando $ pip3 --version. Se necessário, instale o pip para gerenciar suas bibliotecas Python.

Assim como o comando $ python, apenas $ pip instala pacotes Python 2, portanto utilize $ pip3 para o Python 3. O pip torna a instalação de pacotes Python muito mais simples e organizada. Você já pode testá-lo no próximo tópico, instalando o Jupyter.

Jupyter Notebook / JupyterLab

Outra ferramenta muito utilizada para Ciência de Dados é o Jupyter Notebook. Esta ferramenta viabiliza a combinação de código executável em tempo real, textos descritivos (Markdown) e outras visualizações interativas, em um documento com a extensão “*.ipynb”. Instale o Jupyter com o comando:

$ pip3 install jupyter

JupyterLab é a interface que aprimora a experiência dos notebooks, com uma estrutura modular, permitindo abrir vários tipos de arquivos em diferentes guias na mesma janela. Oferecendo uma experiência semelhante ao IDE. Você pode instalar o JupyterLab utilizando o comando:

$ pip3 install jupyterlab

Também é possível ter uma experiência semelhante no VSCodium, há suporte para arquivos “*.ipynb”, gerando uma visualização equivalente no editor de texto.

Para inicializar o Jupyter Lab, é recomendável sua inicialização dentro do seu devido projeto. Por exemplo, se estamos desenvolvendo em um diretório chamado ‘my-project’, na sua pasta Developer, você deve acessar o diretório pelo terminal com o comando $ cd Developer/my-project e em seguida executar $ jupyter lab no seu console. Para utilizar os Notebooks, copie e cole os endereços gerados na mensagem do console: http://localhost:8888/ + token.

Um pouco mais avaçado

Se você já se sente confortável utilizando o terminal no macOS / linux, recomendo dar um upgrade no visual e usabilidade dele com o Oh My Zsh, utilize a instruções deste link.

Também é possível criar um alias para python3 (alias python="python3") e pip3 (alias pip="pip3"). Desta forma você não vai precisar digitar o ‘3’ ao final de cada um dos comandos.
